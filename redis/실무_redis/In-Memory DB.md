# In-Memory DB
왜 Redis를 사용하려고 하는가.

이 의도에는 In-memory에서 사용되는 데이터베이스라는 기준에서 시작한다.

일반적으로 사용하는 MySQL이나 PostgreSQL 과같은 관계형 데이터베이스는 ACID를 추구하여, 데이터간 관계성을 정의하고 이들간의 유기적인 질의를 수행할 수 있도록하는데,

이 말인즉슨, 관계성을 갖는 데이터 환경에서 서로 복잡한 질의에 대해, 명확한 결과를 산출함에 있다고 생각한다.

Redis는 In-Memory 기반의 Key-Value 구조를 가지는 NoSQL 데이터 저장소이다.

때문에, 데이터를 디스크가 아닌 메모리에 상주시키고 처리하기 때문에, 디스크 I/O의 병목 없이 빠르게 결과를 처리한다는 가장 큰 장점을 가지고 있다.

Redis DataBase나 Append Only Files 와 같은 영속적인 옵션을 사용해서 데이터의 소실에 대해 보장할 수는 있긴하다. Failover를 주목적으로한다.
